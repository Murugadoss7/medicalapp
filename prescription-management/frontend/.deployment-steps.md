# Quick Vercel Deployment Steps

## Option 1: Fix Current Web Deployment (Recommended)

1. Add environment variable in Vercel Dashboard:
   - Go to: Settings → Environment Variables
   - Add: `VITE_API_URL` = `http://localhost:8000/api/v1` (temporary)
   - Click Save

2. Redeploy:
   - Go to Deployments tab
   - Click (...) → Redeploy

## Option 2: Deploy via CLI

```bash
# Install Vercel CLI
npm install -g vercel

# Login
vercel login

# Navigate to frontend
cd prescription-management/frontend

# Add environment variable
vercel env add VITE_API_URL production
# When prompted, enter: http://localhost:8000/api/v1

# Deploy to production
vercel --prod
```

## Next Steps After Frontend Deploys

1. Deploy backend to Railway/Render (see DEPLOYMENT_GUIDE.md)
2. Get backend production URL
3. Update VITE_API_URL in Vercel to production backend URL
4. Redeploy frontend
